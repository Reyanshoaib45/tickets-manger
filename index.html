Executive Summary

This document explains how our repair shop software should work, in simple terms. We're building a system where technicians can fix phones, order parts, and managers can see what's happening.

Think of it like a video game where each repair job is a quest, and technicians are the heroes completing quests. This document tells the programmers how to build that game!

Imagine This Like a Game:
  • Technicians are players with their own game boards
  • Repair jobs are missions they need to complete
  • Parts ordering is like sending for special tools
  • The Task Board is where everyone can give each other mini-missions

Designed Simple Like a Game

Easy for everyone to understand

Think of It Like This:

Technician (The Fixer)

Only sees phones they need to fix. Like having their own personal to-do list.

Store Person (The Shopper)

Gets shopping lists (part orders) from technicians. Buys parts online.

Manager (The Boss)

Can see everything but doesn't get confused by too much detail.

Customer (The Waiter)

Only gets important updates: "Your phone is ready!" or "We can't fix it."

What's Most Important?

Priority Levels Explained Simply:

P0 = "Fix this NOW or we can't work!"
P1 = "Very important, do this soon"
P2 = "Nice to have, but can wait"

Level What It Means When to Fix
P0 System is broken! Can't work without this fix. RIGHT NOW!
P1 Important feature not working properly This week
P2 Would be nice to have, makes things easier Next month

How It Should Look

Color Guide:
Red = Waiting for parts (Important!)
Green = Fixed and ready
Orange = Being worked on
Blue = New, not started yet

Detailed Ticket Breakdown

What Are These Tickets?

Each ticket below is like a "to-do" list item for our programmers. They tell them exactly what to build and how it should work.

P1 - IMPORTANT

TICKET #1: Keep Repair Board Separate

Technicians should only see what they need to fix
For Technicians No Confusion
Simple Explanation:

Imagine you're a chef in a kitchen. You only need to see the orders for YOUR station, not every single order in the restaurant. That's what we want for technicians!

What Needs to Happen:
1
Only technicians can see the "Repair Jobs" screen
2
Everyone else (sales, managers) sees only the general "Task Board"
3
If you're not a technician and try to visit the repair page, you get a "Not Found" message
How We Know It's Working:
P0 - SUPER IMPORTANT

TICKET #2: Ordering Parts (Like Sending a Shopping List)

Technician needs a part ‚Üí Creates shopping list for store person
Critical Core Feature
Simple Explanation:

When fixing a phone, if you need a new screen, you write down "iPhone screen from Amazon.com - $50" and give it to the store guy. He buys it, then tells you when it arrives.

Step-by-Step Process:
1
Technician clicks "Order Part" button on a repair job
2
A popup appears asking: Part Link? Price? Who should buy it?
3
System creates a "shopping task" for the store person
4
Repair job turns yellow/orange to show "waiting for parts"
5
Store person buys part, marks task done ‚Üí Technician gets notified
Example:

Task Created: "Buy screen for John's iPhone 12 - Amazon link - $49.99 - Assign to Store Bob"

How We Know It's Working:
P1 - IMPORTANT

TICKET #3: Task Board Rules (Like Classroom Rules)

Who can do what with tasks - Clear rules needed
Rules Permissions
Simple Explanation:

Think of tasks like sticky notes on a board. You can put a sticky note on someone else's desk (assign a task), but you can't move THEIR sticky notes around. Only they can move their own notes.

The Rules:
CAN DO
  • Create tasks for anyone
  • Delete tasks you created
  • Take someone else's task if they're away
  • See who did what and when
CAN'T DO
  • Move other people's tasks
  • Delete tasks you didn't create
  • Hide who created a task
How We Know It's Working:
P1 - IMPORTANT

TICKET #4: Smart Notifications (Only Important Emails)

Send emails for important stuff, not everything
Emails Notifications
Simple Explanation:

You don't want email for every little thing (that's spam!). You only want email when:
1. Someone gives you work to do
2. Your ordered part arrives
3. A customer's phone is ready

Who Gets What Emails:
When This Happens... Who Gets Email Customer Email?
Task assigned to you YOU get email No
Part you ordered arrives Technician gets email No
Phone fixed and ready Manager knows YES - "Come get your phone!"
Phone can't be fixed Manager knows YES - "Sorry, we can't fix it"
Just ordering parts Store person gets task NO - Don't bother customer yet
Important: Customers should NOT get emails every time we order parts. Only when their phone is actually ready or can't be fixed.
How We Know It's Working:
